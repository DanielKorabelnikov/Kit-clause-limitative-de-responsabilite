# Faire un retour sur le guide

Ce guide est volontairement conçu comme un support de travail évolutif. Les retours sont utiles pour identifier les points à préciser, les formulations à améliorer ou les cas pratiques à ajouter.

## Comment faire un retour ?

Vous pouvez réagir de deux manières :

1. **En commentaire du post LinkedIn de publication**  
   C’est le format le plus simple pour partager une observation rapide, une question ou un cas d’usage.

2. **Via GitHub, pour une remarque plus structurée**  
   Vous pouvez proposer une amélioration, signaler un point à clarifier ou suggérer une variante de clause.

[Proposer une remarque sur GitHub](https://github.com/danielkorabelnikov/kit-clause-limitative-responsabilite/issues/new?title=Retour%20sur%20le%20guide%20clause%20limitative%20de%20responsabilit%C3%A9&body=Bonjour%2C%0A%0AVoici%20mon%20retour%20sur%20le%20guide%20%3A%0A%0A-%20Document%20ou%20point%20concern%C3%A9%20%3A%0A-%20Commentaire%20%3A%0A-%20Suggestion%20%C3%A9ventuelle%20%3A%0A%0A)

## Types de retours utiles

- un point qui mérite une clarification ;
- une clause ou variante qui pourrait être ajoutée ;
- un cas d’usage rencontré en pratique ;
- une difficulté fréquente de négociation ;
- une remarque sur la lisibilité du guide.

## Précaution

Merci de ne pas partager d’informations confidentielles, de données personnelles, d’extraits de contrats non publics ou d’éléments permettant d’identifier une situation client. Les retours doivent rester généraux ou anonymisés.
